#### 5.1 ip地址
查看本机ip与主机信息：
```shell
hostname -I #本机ip
ip address #本机ip详细信息
hostname #本机名称
getent hosts #查看本机存储的host信息
```
linux主机通过DHCP服务来获取ip地址。
#### 5.2 网络连接
```shell
ping 8.8.8.8
telnet 192.168.1.1 22 #检查是否能连接到192.168.1.1的22端口
wget --spider http://192.168.1.1 #检查192.168.1.1的http服务是否正常
```
可以使用wget、curl通过url下载文件到当前工作目录
```shell
wget -b http://archive.apache.org/handop-3.3.0.tar.gz #-b代表后台下载

curl https://www.baidu.com #像www.baidu.com发起网络请求并显示请求结果
curl -O http://archive.apache.org/handop-3.3.0.tar.gz #-O代表下载文件
```
#### 5.3 端口
linux中一共有1-65535的端口，分为3大类：
1.公认端口：1-1023，这些端口被常用服务占用或预留使用，不要占用这些端口
2.注册端口：1024-49151，可以用于自己启动的长期服务
3.动态端口：49161之后，用于临时建立连接，用毕即解除占用
```shell
ss -tulnp #列出端口使用情况
```
#### 5.4 进程
使用ps aux查看本机所有进程，每个进程都有一个独特的pid
```shell
ps aux #查看运行中的所有进程
ps aux | grep sshd #查看运行中的sshd进程
kill 911 #杀死pid为911的进程
```
#### 5.5 环境变量
环境变量采用key=value的形式存储，用于存储操作系统中的一些关键信息，例如所有命令的二进制文件的存放目录就存放在环境变量中key为PATH的变量中。可以使用`env`命令查看linux的环境变量。
```shell
env
env | grep PATH #查找PATH环境变量的信息
```
执行命令时，实际上系统先从环境变量PATH中找出所有可执行文件的所在目录，再依次从这些目录中找出对应的可执行文件。如果这些目录中都找不到对应的可执行文件则报错未找到命令。

`$`符号可以快速获取一个变量，可以将变量名用{}包裹，常与echo搭配使用，打印出变量名所代表的值
```shell
echo ${PATH} #相当于执行env | grep PATH
VARIABLE="Alice"
echo ${VARIABLE} #打印出Alice
```
##### 5.5.1 设置环境变量
快速设置一个临时环境变量，shell关闭后消失：
```shell
export PRICE="1000y"
```
当前用户的环境变量保存在`~/.bashrc`中，快速设置一个永久的当前用户环境变量：
```shell
echo 'export MY_APP_ENV=production' >> ~/.bashrc #设置一个key为MY_APP_EVN，value为production的环境变量到当前用户
source ~/.bashrc #重新加载配置
echo ${MY_APP_ENV} #打印出环境变量MY_APP_EVN
```
